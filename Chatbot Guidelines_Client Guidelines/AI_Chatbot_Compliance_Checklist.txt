INTRODUCTION
────────────────────────────────────────────────────────────────
This checklist helps organizations ensure their AI-powered
chatbot deployments meet international compliance standards.
It covers six major compliance frameworks relevant to
Canadian businesses and their global clients.

Before deploying any AI chatbot in a production environment,
review each section and confirm all applicable items are
addressed. Items marked MANDATORY must be completed before
launch. Items marked RECOMMENDED are best practices that
significantly reduce risk and liability.


════════════════════════════════════════════════════════════════
SECTION 1 — PIPEDA COMPLIANCE
Personal Information Protection and Electronic Documents Act
Jurisdiction: Canada — MANDATORY for all Canadian clients
════════════════════════════════════════════════════════════════

[ ] MANDATORY — Privacy Notice displayed on chatbot interface
    Users must be informed what data is collected and why
    before they interact with the chatbot.

[ ] MANDATORY — Consent obtained before document processing
    Users must explicitly agree to having their documents
    processed by the AI system before uploading.

[ ] MANDATORY — Purpose limitation clearly stated
    The chatbot must only process documents for the stated
    purpose. Documents uploaded for one purpose must not
    be used for another.

[ ] MANDATORY — Data minimization practiced
    Only collect and process the minimum data necessary
    to answer the user's question. Do not store entire
    documents beyond the active session.

[ ] MANDATORY — Session data cleared after use
    All uploaded documents and conversation history must
    be cleared when the user session ends. No permanent
    storage of user documents.

[ ] MANDATORY — Contact information provided
    Users must be given a way to contact the organization
    regarding their data and privacy rights.

[ ] RECOMMENDED — Data retention policy documented
    Clearly document how long any session data is retained
    and when it is automatically deleted.

[ ] RECOMMENDED — Privacy policy linked in the UI
    Provide a link to the full organizational privacy policy
    within the chatbot interface.


════════════════════════════════════════════════════════════════
SECTION 2 — GDPR COMPLIANCE
General Data Protection Regulation
Jurisdiction: European Union — MANDATORY if serving EU users
════════════════════════════════════════════════════════════════

[ ] MANDATORY — Legal basis for processing documented
    Identify and document the legal basis for processing
    personal data (consent, legitimate interest, contract).

[ ] MANDATORY — Right to erasure supported
    Users must be able to request deletion of their data
    at any time. Provide a clear mechanism to do this.

[ ] MANDATORY — Data processing transparency
    Inform users clearly that their documents are being
    processed by an AI system and identify which AI
    provider is being used (Groq, Google, etc.).

[ ] MANDATORY — Cross border data transfer disclosure
    If user data is sent to servers outside the EU (such
    as US based LLM providers) this must be disclosed
    and appropriate safeguards documented.

[ ] MANDATORY — Data breach notification process
    Have a documented process for notifying users and
    authorities within 72 hours of a data breach.

[ ] RECOMMENDED — Data Protection Impact Assessment
    Complete a DPIA before deploying the chatbot especially
    if processing sensitive personal information.

[ ] RECOMMENDED — EU representative appointed
    If the organization is outside the EU but serves EU
    users appoint an EU representative as required by GDPR.

[ ] RECOMMENDED — Cookie consent implemented
    If the chatbot uses cookies or tracking implement
    a proper consent mechanism.


════════════════════════════════════════════════════════════════
SECTION 3 — ISO/IEC 42001 COMPLIANCE
AI Management System Standard
Jurisdiction: International — RECOMMENDED for enterprise AI
════════════════════════════════════════════════════════════════

[ ] MANDATORY — AI system purpose documented
    Clearly document what the AI chatbot is designed to do,
    what it is not designed to do, and its known limitations.

[ ] MANDATORY — AI transparency disclosure
    Users must be informed they are interacting with an AI
    system and not a human. This must be clear and prominent
    in the interface.

[ ] MANDATORY — Human oversight mechanism in place
    Define how humans will monitor, review, and if necessary
    override AI outputs. Document the escalation process.

[ ] MANDATORY — Risk assessment completed
    Identify and document potential risks of the AI system
    including incorrect answers, bias, and misuse scenarios.

[ ] RECOMMENDED — AI system performance monitored
    Implement logging and monitoring to track chatbot
    performance, accuracy, and user satisfaction over time.

[ ] RECOMMENDED — Regular audits scheduled
    Schedule quarterly reviews of AI system outputs to
    identify and correct any patterns of incorrect or
    biased responses.

[ ] RECOMMENDED — Incident response plan documented
    Have a clear plan for what to do when the AI system
    produces harmful, incorrect, or inappropriate outputs.

[ ] RECOMMENDED — Stakeholder communication plan
    Document how AI system changes, updates, and incidents
    will be communicated to relevant stakeholders.


════════════════════════════════════════════════════════════════
SECTION 4 — ISO/IEC 27001 COMPLIANCE
Information Security Management System
Jurisdiction: International — MANDATORY for enterprise clients
════════════════════════════════════════════════════════════════

[ ] MANDATORY — API keys stored securely
    All API keys must be stored in environment variables
    or a secrets manager. Never hardcoded in source code
    or stored in version control systems like GitHub.

[ ] MANDATORY — HTTPS encryption enforced
    All data transmission between users and the chatbot
    must be encrypted using HTTPS. Never deploy on HTTP.

[ ] MANDATORY — Access control implemented
    Restrict access to the chatbot administration panel
    and deployment environment to authorized personnel only.

[ ] MANDATORY — Dependency vulnerability scanning
    Regularly scan Python packages and dependencies for
    known security vulnerabilities and update promptly.

[ ] MANDATORY — Secure development practices followed
    Follow secure coding guidelines. Never commit secrets
    to version control. Use .gitignore properly.

[ ] RECOMMENDED — Penetration testing conducted
    Commission a security penetration test before deploying
    to enterprise clients handling sensitive documents.

[ ] RECOMMENDED — Security incident log maintained
    Keep a log of all security incidents, near misses,
    and remediation actions taken.

[ ] RECOMMENDED — Third party security assessment
    For high value clients obtain a third party security
    assessment of the complete deployment environment.


════════════════════════════════════════════════════════════════
SECTION 5 — HIPAA COMPLIANCE
Health Insurance Portability and Accountability Act
Jurisdiction: United States — MANDATORY for healthcare clients
════════════════════════════════════════════════════════════════

[ ] MANDATORY — PHI identification and protection
    Identify whether uploaded documents may contain
    Protected Health Information (PHI) and implement
    appropriate safeguards.

[ ] MANDATORY — Business Associate Agreement signed
    Sign a BAA with all third party service providers
    that process PHI including LLM API providers.

[ ] MANDATORY — Minimum necessary standard applied
    Only process the minimum PHI necessary to respond
    to the user's specific question.

[ ] MANDATORY — Audit controls implemented
    Implement logging of all access to systems containing
    PHI including chatbot queries involving health data.

[ ] MANDATORY — Transmission security enforced
    All PHI transmitted to and from the chatbot must be
    encrypted in transit using industry standard protocols.

[ ] RECOMMENDED — Staff training completed
    All staff with access to the chatbot system must
    complete HIPAA compliance training annually.

[ ] RECOMMENDED — Risk analysis documented
    Complete and document a thorough HIPAA risk analysis
    before handling any health related documents.


════════════════════════════════════════════════════════════════
SECTION 6 — SOC 2 COMPLIANCE
Service Organization Control 2
Jurisdiction: United States — RECOMMENDED for SaaS clients
════════════════════════════════════════════════════════════════

[ ] RECOMMENDED — Security controls documented
    Document all security controls in place for the
    chatbot system including access controls, encryption,
    and monitoring procedures.

[ ] RECOMMENDED — Availability SLA defined
    Define and document the expected uptime and
    availability of the chatbot service for clients.

[ ] RECOMMENDED — Confidentiality controls implemented
    Document how client documents and conversation data
    are kept confidential and separated from other clients.

[ ] RECOMMENDED — Processing integrity verified
    Implement checks to ensure the chatbot processes
    documents completely, accurately, and as intended.

[ ] RECOMMENDED — Change management process documented
    Document how updates, patches, and changes to the
    chatbot system are tested and deployed safely.


════════════════════════════════════════════════════════════════
SECTION 7 — GENERAL AI ETHICS CHECKLIST
Applicable to all deployments regardless of jurisdiction
════════════════════════════════════════════════════════════════

[ ] MANDATORY — Hallucination prevention implemented
    The chatbot must be instructed to only answer from
    document content and clearly state when it cannot
    find an answer rather than inventing one.

[ ] MANDATORY — Bias awareness documented
    Acknowledge that LLMs may contain biases and document
    how this risk is managed in the specific use case.

[ ] MANDATORY — Accessibility considered
    Ensure the chatbot interface is accessible to users
    with disabilities following WCAG 2.1 guidelines.

[ ] MANDATORY — Clear AI identification
    Never allow the chatbot to claim to be human or
    misrepresent itself as anything other than an AI.

[ ] RECOMMENDED — Feedback mechanism provided
    Give users a way to report incorrect or harmful
    chatbot responses so they can be reviewed and fixed.

[ ] RECOMMENDED — Fairness testing conducted
    Test the chatbot with diverse user groups and document
    types to ensure consistent and fair performance.

[ ] RECOMMENDED — Environmental impact considered
    Document the carbon footprint of LLM API usage and
    consider this in the overall sustainability strategy.

MINIMUM VIABLE COMPLIANCE FOR CANADIAN CLIENTS
Quick Start Checklist
════════════════════════════════════════════════════════════════

If you need to deploy quickly for a Canadian client ensure
these absolute minimum items are completed first:

[ ] 1. Privacy notice displayed in the UI
[ ] 2. User consent obtained before document upload
[ ] 3. Session data cleared when session ends
[ ] 4. AI system clearly identified to users
[ ] 5. API keys stored securely in environment variables
[ ] 6. HTTPS encryption enforced on deployment
[ ] 7. Hallucination prevention prompt implemented
[ ] 8. Contact information provided for data requests
[ ] 9. .env file excluded from GitHub via .gitignore
[ ] 10. Purpose and limitations of AI clearly documented
